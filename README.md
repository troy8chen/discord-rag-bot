## üèóÔ∏è Architecture Overview

This Discord bot works with a **separate RAG service** to provide expert-level assistance:

```mermaid
graph TB
    A[Discord User] --> B[Discord Bot<br/>troy8chen/discord-rag-bot]
    B --> C[Redis Message Queue]
    C --> D[RAG Worker<br/>troy8chen/tech-docs]
    D --> E[AI Knowledge Base<br/>Pinecone + OpenAI]
    E --> D
    D --> C
    C --> B
    B --> A
    
    style A fill:#7289DA
    style B fill:#5865F2
    style C fill:#DC382D
    style D fill:#00D924
    style E fill:#FF6B6B
```

### üîÑ Event-Driven Flow
1. **User sends message** ‚Üí Discord bot receives interaction
2. **Bot publishes query** ‚Üí Redis `rag:query` channel
3. **RAG worker processes** ‚Üí AI generates expert response
4. **Worker publishes response** ‚Üí Redis `rag:response` channel  
5. **Bot receives response** ‚Üí Formatted Discord message sent

#### ü§ñ Discord Bot Configuration

1. **Create Discord Application**
   - Go to [Discord Developer Portal](https://discord.com/developers/applications)
   - Click "New Application" ‚Üí Name your bot
   - Navigate to "Bot" section ‚Üí Reset Token (save this as `DISCORD_BOT_TOKEN`)
   - Copy Application ID (save this as `DISCORD_CLIENT_ID`)

2. **Configure Bot Permissions**
   ```
   Required Permissions:
   ‚úÖ Send Messages
   ‚úÖ Use Slash Commands  
   ‚úÖ Read Message History
   ‚úÖ Embed Links
   ‚úÖ Add Reactions
   ```

3. **Invite Bot to Server**
   ```
   https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=277025573888&scope=bot%20applications.commands
   ```
   Replace `YOUR_CLIENT_ID` with your actual client ID.

### üéØ Redis Deployment Strategy

**Why Deploy Redis Separately?**

‚úÖ **Shared Resource**: Both `discord-rag-bot` AND `tech-docs` repos use same Redis  
‚úÖ **Independent Scaling**: Scale Redis separately from apps  
‚úÖ **Cost Efficient**: One Redis instance serves multiple services  
‚úÖ **Better Reliability**: Managed services have built-in backups/monitoring  
‚úÖ **Microservices Best Practice**: Proper separation of concerns  

**Architecture:**
```mermaid
graph TB
    A[Discord Bot<br/>Railway/Docker] --> D[Redis Cloud<br/>Managed Service]
    B[RAG Worker<br/>Railway/Docker] --> D
    C[Next.js Web<br/>Vercel/Railway] --> D
    
    style D fill:#ff6b6b,color:#fff
    style A fill:#e1f5fe
    style B fill:#f3e5f5
    style C fill:#e8f5e8
```

**Setup Instructions:**

1. **Create Redis Cloud Account**: [redis.com/try-free](https://redis.com/try-free/)
2. **Create Database**:
   - Choose "Fixed" plan (30MB free)
   - Select region closest to your deployments
   - Note the connection string
3. **Get Connection URL**:
   ```
   Format: redis://username:password@host:port
   Example: redis://default:abc123@redis-12345.c1.us-east-1-1.ec2.cloud.redislabs.com:12345
   ```
4. **Update Both Projects**:
   ```bash
   # discord-rag-bot/.env
   REDIS_URL=redis://username:password@host:port
   
   # tech-docs/.env.local  
   REDIS_URL=redis://username:password@host:port
   ```

**Deployment Benefits:**
- üîÑ **Zero Downtime**: Update apps without affecting Redis
- üìä **Monitoring**: Built-in Redis metrics and alerts
- üîß **Maintenance**: Automatic backups and updates
- üí∞ **Cost**: Free tier supports 1000+ daily queries
